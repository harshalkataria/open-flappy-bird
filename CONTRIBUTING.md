# Contributing to Open Flappy Bird

Thank you for your interest in contributing to Open Flappy Bird! This document provides guidelines and instructions for contributing.

## Code of Conduct

By participating in this project, you agree to maintain a respectful and inclusive environment for everyone.

## How Can I Contribute?

### Reporting Bugs

- Check if the bug has already been reported in the Issues section
- Use the bug report template when creating a new issue
- Include detailed steps to reproduce the bug
- Include screenshots if applicable

### Suggesting Features

- Check if the feature has already been suggested in the Issues section
- Use the feature request template when creating a new issue
- Provide a clear description of the feature and its benefits

### Pull Requests

1. Fork the repository
2. Create a new branch for your feature or bugfix
3. Make your changes
4. Run tests to ensure your changes don't break existing functionality
5. Submit a pull request

## Development Setup

1. Clone the repository
2. Open the project in Unity (recommended version: 2021.3 LTS)
3. Make sure all dependencies are installed

## Coding Standards

- Follow C# coding conventions
- Use meaningful variable and function names
- Write comments for complex logic
- Follow SOLID principles
- Include XML documentation for public methods

## Git Workflow

- Use feature branches for new features
- Use bugfix branches for bug fixes
- Keep commits small and focused
- Write clear commit messages

## Testing

- Test your changes thoroughly before submitting a pull request
- Include unit tests for new functionality when possible

## Documentation

- Update documentation to reflect your changes
- Document public APIs and important functionality

Thank you for contributing to Open Flappy Bird!
